# MVP 定义

[English](./MVP.md)

## 项目目标

构建一个基于 Python、面向个人使用的 A 股量化可视化分析系统。

第一版主要回答这些问题：

- `上证指数` 和 `沪深300` 在中长期维度上表现如何？
- 选定的代表性个股相对于这两个基准指数表现怎样？
- 能否用简单且可解释的方式总结指数与个股的长期趋势状态？

## MVP 必须完成的内容

### 1. 数据采集

系统需要能够采集并本地保存：

- 选定指数的基础信息
- 跟踪指数的日线历史行情
- 观察样本个股的基础信息
- 观察样本个股的日线历史行情

### 2. 指数分析

系统需要提供：

- 指数长期价格走势图
- 指数均线结构
- 20、60、120 个交易日等时间窗口收益率
- 指数的简单趋势标签

### 3. 个股长期趋势分析

系统需要提供：

- 个股长期价格走势图
- MA20、MA60、MA120、MA250 等均线系统
- 成交量趋势辅助
- 相对基准指数的强弱表现
- `上升趋势`、`震荡`、`走弱` 等趋势标签

### 4. 可视化输出

在网页出现之前，MVP 需要先支持可视化输出。

第一阶段可接受的形式：

- Jupyter Notebook
- Python 图表脚本
- 本地 HTML 图表导出
- 报告使用的静态图片输出

## 已选择的公开数据源

当前主数据源为 `efinance`，`TickFlow` 作为备用源。

当前选择依据是其文档中已明确支持：

- 通过 `ef.stock.get_quote_history()` 获取股票历史行情
- 通过 `ef.stock.get_base_info()` 获取个股基础信息
- 通过 `ef.stock.get_realtime_quotes()` 获取指数相关行情快照

## MVP 暂不包含的内容

第一版明确不做：

- 板块和行业分析
- 未来几天的短线价格预测
- 高频或分时级分析
- 自动交易
- 完整回测系统
- 用户登录或多用户支持
- 新闻情绪分析
- 复杂机器学习模型

## 第一阶段观察对象

第一阶段跟踪的指数：

- 上证指数
- 沪深300

建议的第一批候选个股：

- `600519` 贵州茅台
- `600036` 招商银行
- `601318` 中国平安
- `600900` 长江电力
- `601899` 紫金矿业

## MVP 输出形式

第一版至少应产出：

1. 每个跟踪指数至少一张长期趋势图
2. 每只选定个股至少一张长期趋势图
3. 一份可解释的指数与个股趋势总结
4. 一个可重复生成这些结果的 notebook 或脚本

## 成功标准

满足以下条件，就说明 MVP 跑通了：

1. 能用一个命令或一个脚本完成本地数据更新
2. 能稳定重复计算指数和个股分析结果
3. 能生成清晰展示长期趋势结构的可视化结果
4. 能解释为什么某个指数或个股被标记为强、弱或中性

## 数据优先原则

先不要建网页。

推荐顺序是：

1. 数据采集
2. 数据存储
3. 指标计算
4. 可视化输出
5. 本地网页
6. Azure 部署
