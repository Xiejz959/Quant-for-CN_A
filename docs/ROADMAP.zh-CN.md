# 开发路线图

[English](./ROADMAP.md)

## Phase 0: 范围冻结

目标：
先把项目边界定住，避免范围失控。

产出：

- 清晰的 MVP 定义
- 第一版功能列表
- 明确的非目标
- 以 Python 为中心的技术方向

完成标志：
你能够用一句话描述项目，并且知道第一版先不做什么。

## Phase 1: 数据源调研

目标：
选定支持 A 股指数与个股趋势分析的公开数据源。

核心问题：

- 哪个来源提供指数数据？
- 哪个来源提供个股历史行情？
- 哪个来源提供个股基础信息？
- 哪个来源对学生项目足够稳定？

产出：

- 数据源对比结论
- 最终选定的数据源
- 所需字段清单

完成标志：
你明确知道每类核心数据从哪里来。

## Phase 2: 本地数据管道

目标：
建立可重复执行的本地数据更新流程。

产出：

- SQLite 数据表设计
- 数据下载脚本
- 指数和个股数据更新流程
- 缺失值与重复值检查规则

完成标志：
你可以不靠手工修改就刷新核心数据。

## Phase 3: 分析引擎

目标：
把原始市场数据转成有意义的指数与个股趋势信号。

产出：

- 指数多窗口收益率逻辑
- 指数均线结构分析
- 个股均线与相对强弱指标
- 基于规则的趋势标签

完成标志：
你可以从本地数据稳定计算出分析结果。

## Phase 4: 可视化层

目标：
在做网页之前，先让分析结果能够被直观看到和理解。

产出：

- 指数趋势图
- 个股长期趋势图
- 简洁的报告 notebook 或导出脚本

完成标志：
你可以不去翻原始表格，也能看懂趋势结果。

## Phase 5: 本地应用层

目标：
等分析稳定后，再把数据和图表包成 `localhost` 应用。

推荐起步方案：

- Streamlit

未来升级方案：

- FastAPI 加独立前端

完成标志：
你已经信任数据和图表，只差一个更好的交互入口。

## Phase 6: Azure 部署

目标：
把成熟的本地项目部署到云端。

产出：

- 可部署的项目结构
- 环境配置
- 如有需要的云端数据库方案
- Azure 部署流程

完成标志：
项目可以脱离你的本地机器稳定运行。
