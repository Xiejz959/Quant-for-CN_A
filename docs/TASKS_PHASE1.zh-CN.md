# 第一阶段任务拆解

[English](./TASKS_PHASE1.md)

## 当前目标

在做任何网页之前，先完成指数和个股的数据与可视化 MVP。

## 第 1 周：项目准备与数据源确认

任务：

- 确认基础目录结构
- 确认主公开数据源
- 列出跟踪指数和候选个股所需字段
- 明确数据更新频率假设

完成标志：

- 你已经知道要下载什么数据、从哪里下载、多久更新一次

## 第 2 周：数据库与采集

任务：

- 设计 SQLite 数据表
- 建立个股基础信息采集
- 建立个股日线行情采集
- 建立指数行情采集

完成标志：

- 你可以把指数和个股核心数据写入本地存储

## 第 3 周：核心分析

任务：

- 计算指数多窗口收益率
- 计算指数均线结构
- 计算个股均线
- 计算个股相对强弱
- 定义可解释的趋势标签

完成标志：

- 你已经得到稳定且可解释的分析结果表

## 第 4 周：可视化输出

任务：

- 生成指数趋势图
- 生成个股长期趋势图
- 导出一个可复用的 notebook 或报告脚本

完成标志：

- 你可以直接通过本地输出观察指数与个股趋势

## 最低交付清单

第一版最小可交付成果应包括：

- 一个刷新数据的命令或脚本
- 一个 SQLite 数据库文件
- 一个输出指数与个股趋势结果的分析脚本
- 一个可视化脚本或 notebook
- 一个指数样例输出和一个个股样例输出

## 建议的实现顺序

1. 指数基础信息和个股基础信息
2. 指数与个股日线历史行情
3. 指数趋势计算
4. 个股趋势计算
5. 可视化输出

## 建议优先实现的指标

优先使用简单且可解释的指标：

- 5 日收益率
- 20 日收益率
- 60 日收益率
- 120 日收益率
- 均线斜率
- 价格是否站上长期均线
- 相对上证指数或沪深300的强弱

## 重要原则

- 优先保证数据管道稳定，而不是 UI 漂亮
- 在描述性分析可信之前，不要做复杂预测
- 保存中间结果，保证分析可复现
- 分析脚本与展示逻辑保持解耦
